# Design Category Guidelines

## 1. Landing Page
**Purpose**: introduce a product or service and convert visitors.

**Required elements**
- Hero section with strong value proposition
- Clear CTA buttons (Get Started, Sign Up, etc.)
- Features/benefits list (3–6 items)
- Social proof (logos, stats, testimonials)
- Footer with contact links

**Design traits**
- Bold typography and gradients
- Straightforward storytelling
- Vertical flow that supports scrolling
- Responsive across breakpoints

**Layout ideas**
1. Centered hero
2. Split-screen (copy + media/form)
3. Video background
4. Asymmetric storytelling
5. Minimalist stack

---

## 2. Dashboard
**Purpose**: visualize and manage product data.

**Required elements**
- Sidebar or top navigation
- KPI/stat cards
- Charts/graphs
- Tables (orders, users, etc.)
- Activity feed
- Filters/search controls

**Design traits**
- Clear information hierarchy
- Scannable density
- Status colors for success/warning/error

**Layout ideas**
1. Sidebar + content grid
2. Top nav + tabs
3. Card-based masonry

---

## 3. E-commerce
**Purpose**: merchandise products and streamline checkout.

**Required elements**
- Product grid/cards with pricing
- Filters and sort controls
- Cart icon with count
- Product detail view (gallery, price, reviews)
- Prominent “Add to cart” actions

**Design traits**
- Image-first presentation
- Clear price and rating indicators
- Trust signals (shipping, security, support)

**Layout ideas**
1. 3–4 column grid
2. Detail page with gallery + purchase info
3. Checkout/summary steps

---

## 4. Portfolio
**Purpose**: showcase work, craft, or case studies.

**Required elements**
- Project gallery/cards
- Project details (images, description, stack)
- About/intro section
- Contact or social links

**Design traits**
- Large imagery and generous whitespace
- Minimal copy
- Expressive layouts

**Layout ideas**
1. Masonry grid
2. Fullscreen carousel
3. Grid with hover reveal

---

## 5. Blog / Content Hub
**Purpose**: deliver editorial content clearly.

**Required elements**
- Post list/cards with excerpt and metadata
- Article header (title, date, author, category)
- Body typography styles
- Sidebar (categories, featured posts, tags)
- Comments or reactions (optional)

**Design traits**
- Readability first: line length, spacing, contrast
- Balanced text + imagery
- Clear content hierarchy

**Layout ideas**
1. 2–3 column grid
2. Magazine-style featured hero + list
3. Linear list with thumbnail + copy

---

## 6. Component Library
**Purpose**: present reusable UI primitives.

**Required elements**
- Button variants (primary, secondary, outline, icon)
- Form inputs (text, select, checkbox, radio)
- Cards (basic, stat, media)
- Navigation (navbar, tabs, breadcrumbs)
- Alerts/notifications
- Modals/dialogs

**Design traits**
- Consistent spacing, sizing, color ramps
- Explicit interactive states (hover, focus, disabled)
- Accessibility-minded labeling

**Layout ideas**
1. Showcase grid of components
2. Storybook-style stacked sections
3. Interactive playground/demo panel

---

## Color guidance

- **Primary**: main CTA buttons and key links
- **Secondary**: supporting actions/background accents
- **Accent**: badges, highlights, emphasis details

## Typography guidance

- **Headings**: 64–96 px, bold weight
- **Body**: 16–20 px, comfortable line height
- **Labels**: 12–14 px for UI controls and metadata
